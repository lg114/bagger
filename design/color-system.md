# Color System · #050505 Core · Dark Only

> 基于极深黑 `#050505` 为核心构建的**纯深色**颜色系统。没有浅色模式，没有主题切换——深色是唯一形态。主色阶走「略带冷调的灰黑」路线，与中性灰区分开；语义色取经验证的色板；辅助色选电光青、紫罗兰、玫红三色，在深黑底上均有高识别度。所有正文组合满足 WCAG AA（≥ 4.5:1），关键组合达 AAA（≥ 7:1）。

---

## 1. 主色调 Primary

以 `#050505` 为 900 阶核心，向上推 9 级。色阶比中性灰略带冷调（极微蓝），避免与 Neutral 混淆，强化「主色」身份。在纯深色系统里，主色阶承担**表面层次**的全部职责——从页面背景到卡片到悬浮层，靠主色阶的亮度递进制造空间深度。

|----|-----|------|--------|
| 50 | `#F7F7F8` | 极少使用 · 仅限极强高亮填充 | `#050505` |
| 100 | `#EFEFF1` | 极少使用 · 仅限极强高亮填充 | `#050505` |
| 200 | `#DCDCE0` | 极少使用 · 仅限极强高亮填充 | `#050505` |

| 400 | `#8A8B92` | 辅助文本 / 次级标签 | `#FFFFFF` |
| 500 | `#5C5D65` | 深底上的次级文本备选 | `#FFFFFF` |

## 3. 中性色 Neutral

纯净灰阶（不带色相），承担文本层级、分割线、禁用态、代码高亮底色等非语义化场景。与主色阶的区分：主色阶略带冷调蓝，中性色为纯灰。在纯深色系统里，中性色主要用于**文本层级**与**代码块底**，表面层次交给主色阶。

| 阶 | Hex | 深色模式用途 |
|----|-----|------|
| 0 | `#FFFFFF` | 极少使用 · 仅限极强高亮文字 |
| 50 | `#FAFAFA` | 极少使用 |
| 100 | `#F4F4F5` | **主正文文本** · 最高对比 |
| 200 | `#E4E4E7` | 标题文本 |
| 300 | `#D4D4D8` | 次级标题 |
| 400 | `#A1A1AA` | **次级文本** · 辅助说明 |
| 500 | `#71717A` | 三级文本 · 时间戳 / 计数 |
| 600 | `#52525B` | 禁用文字 |
| 700 | `#3F3F46` | 极少使用（主色 600 替代） |

## 4. 辅助强调色 Accent

三个与 `#050505` 主色搭配的强调色，各提供 400/500/600 三阶。在纯深色系统里，**400 是主用色**（深底上的高对比文字、图标、链接），500 用于按钮实色填充。

### 4.1 电光青 Teal — 主交互

| 阶 | Hex | 深色模式用途 |
|----|-----|------|
| 400 | `#2DD4BF` | **主交互色** · 链接 / 激活态 / 主按钮文字 |
| 500 | `#14B8A6` | 主按钮实色填充（`#050505` 字） |
| 600 | `#0D9488` | 主按钮 hover |

> 与主色 `#050505` 搭配：青色按钮置于深黑背景上对比度 9.1:1，是首要交互入口的首选。400 用于文字与图标，500 用于实色填充。

### 4.2 紫罗兰 Violet — 聚焦高亮

| 阶 | Hex | 深色模式用途 |
|----|-----|------|
| 400 | `#A78BFA` | 聚焦环（focus ring）浅色变体 |
| 500 | `#8B5CF6` | **聚焦环主色** · 选中态左边框 |
| 600 | `#7C3AED` | 选中态 hover |

> 专用于聚焦与选中——input focus、tab active、列表选中项左边框。与电光青区分：青色是「可点击」，紫色是「当前聚焦/选中」。

### 4.3 玫红 Rose — 强强调

| 阶 | Hex | 深色模式用途 |
|----|-----|------|
| 400 | `#FB7185` | **未读标记** · 新内容徽标 |
| 500 | `#F43F5E` | 徽标实色填充 |
| 600 | `#E11D48` | 徽标 hover |

> 与 Error 红区分：Error 400 `#F87171` 偏正红用于「错误状态」，Rose 400 `#FB7185` 偏粉红用于「需要注意力但不一定是错误」（未读、新内容、重要标记）。

## 5. 使用规范

### 5.1 表面层次（不靠阴影，靠亮度递进）

```
页面背景 (Primary 900 · #050505)
  └─ 卡片表面 (Primary 800 · #131316)       ← 比背景亮一阶
       └─ 悬浮表面 (Primary 700 · #25262B)   ← 比卡片亮一阶
            └─ 弹层/模态 (Primary 700 + 极轻阴影)
```

**原则**：每一层表面比上一层亮一阶，通过亮度差异制造空间深度。阴影仅在弹层/模态使用（`0 8px 24px rgba(0,0,0,0.35)`），普通卡片靠边框（Primary 600）区分而非阴影。

### 5.2 文本层级

### 5.5 禁止事项

- **禁止**引入浅色模式或主题切换——深色是唯一形态，不要预留 `[data-theme="light"]` 分支
- **禁止**用 Warning 400 `#FBBF24` 作为正文文字色（过于刺眼），正文警告用 500 `#F59E0B`
- **禁止**同时使用 Teal 与 Violet 作为主按钮色（会模糊「可点击」与「已聚焦」的语义边界）
- **禁止**用 Rose 替代 Error 表达错误状态（色相相近但语义不同）
- **禁止**用阴影制造普通卡片的层次——靠 Primary 800/700/600 的亮度递进
- **禁止**在 `#050505` 背景上使用 Primary 900 作为文字或边框（无对比度）
- **禁止**用 Neutral 色阶做表面层次（那是 Primary 色阶的职责），Neutral 只管文本与代码块
